Frequently Asked Questions

{% endblock %} {% block content %}
  • Q. How do I log in?
    A. Log in with your Gavilan student ID (G00 number) here.

  • Q. Do I have to submit before the due date?
    A. It is possible to submit your program at any time, but your instructor does see if and when you submit.

  • Q. How many submissions can I make?
    A. You may make up to 10 submissions per assignment.

  • Q. Can I undo a submission that I've already confirmed?
    A. No, you can only cancel unconfirmed submissions.

  • Q. Is the percentage grade I get for a submission official?
    A. The percentage grade is not official, and does not directly affect your actual grade. It is a quick indicator of how many test cases your program passed.

  • Q. Some of the test cases don't have any input, is this normal?
    A. Normally, yes. Many test cases do not require input. However, if a test case seems like it should have input but does not, let your instructor know.

  • Q. Why doesn't the input for my program show up in the output like when I run it on my own computer?
    A. The Autograder only checks what your program actually outputs itself. The input isn't part of that, so it does not show up in the test results. It is normal for the output to be different from what you might see when running the program in a terminal.

  • Q. What is the "\n" that keeps showing up in test cases and in my program's output?
    A. "\n" is a visual representation of the new line character. It helps show, for example, if there are blank spaces at the end of a line, or if there is a new line character at the end of a program's output. Both of these things would be hard to know without seeing where the new line characters are. It can be differentiated from the literal "\" and "n" characters by its light color and border.

Contact your instructor if you have any other questions.
